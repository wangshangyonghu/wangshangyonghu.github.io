GitHub 访问异常？修改 HOSTS 临时解决教程

简单、免费、无需软件 | 临时应急使用

一、这个方法有用吗？

✅ 能轻度改善 GitHub 打开速度、网页加载失败问题

⚠️ 不能彻底解决克隆慢、大文件下载慢

🔄 IP 会过期，失效后重新换 IP 即可

二、最新可用 GitHub HOSTS 配置

直接复制下面内容，添加到你的 hosts 文件最底部：

# GitHub Hosts（2026可用） 140.82.116.3 github.com 140.82.116.4 github.com 20.205.243.166 github.com 185.199.108.133 raw.githubusercontent.com 185.199.109.133 raw.githubusercontent.com
小提示： 如果失效，请到 ip33.com/dns.html 重新查询最新 IP。

三、如何查询最新可用IP（详细教程）

1. 打开查询网站

点击进入：ip33.com/dns.html（新窗口打开）

2. 输入需要查询的域名

在页面输入框中分别输入以下两个域名：

github.com

raw.githubusercontent.com

3. 点击查询按钮

页面会显示多个 A 记录 IP 地址

4. 复制可用的 IP

挑选 2–3 个响应速度快的 IP，复制下来替换 hosts 文件里旧的 IP 即可

四、Windows 系统修改教程

1. 打开 hosts 文件所在目录

复制下面路径，粘贴到文件夹地址栏 → 回车：

C:\Windows\System32\drivers\etc

2. 用记事本打开 hosts

找到 hosts 文件 → 右键 → 打开方式 → 记事本

3. 在文件最底部粘贴配置

把上面的 HOSTS 代码粘贴到最后一行

4. 保存文件

文件 → 保存

权限不足？
右键记事本 → 以管理员身份运行 → 再打开 hosts 保存即可。

五、立即生效方法

修改完后，刷新网页或重启浏览器即可生效。

如果不生效：

1. 按下 Win + R

2. 输入 ipconfig /flushdns 回车
