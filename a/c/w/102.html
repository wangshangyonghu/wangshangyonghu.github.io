在防火墙“高级设置”中添加IIS服务允许规则

1打开控制面板

按下键盘上的 Win + R 组合键，弹出“运行”窗口。
在“运行”窗口的输入框中，输入 control panel，然后按下 Enter 键，即可打开“控制面板”。

2进入Windows Defender防火墙

在“控制面板”界面中，找到并点击 系统和安全 选项。
在“系统和安全”页面中，继续选择并点击 Windows Defender 防火墙。

3打开高级设置

进入“Windows Defender 防火墙”主界面后，在左侧的菜单列表中，点击 高级设置 选项。
点击后会弹出“高级安全 Windows 防火墙”窗口，此窗口用于配置更细致的防火墙规则。

4选择入站规则

在“高级安全 Windows 防火墙”窗口的左侧栏中，找到并选择 入站规则 选项。
选择后，右侧栏会显示当前系统中已有的入站规则列表。

5启动新建规则向导

在“入站规则”界面的右侧栏中，点击 新建规则 按钮。
点击后会弹出“新建入站规则向导”窗口，后续将通过该向导配置IIS服务的允许规则。

6选择规则类型并设置端口

在“新建入站规则向导”窗口中，规则类型选择 端口，然后点击 下一步。
进入端口设置页面后，选中 TCP 协议，同时勾选 特定本地端口。
在“特定本地端口”对应的文本框中，输入IIS服务需要开放的端口号：
  • HTTP协议默认端口：80
  • HTTPS协议默认端口：443（若使用HTTPS需同时添加）
输入完成后，点击 下一步。
提示：若同时使用HTTP和HTTPS，可在文本框中用英文逗号分隔两个端口号，如“80,443”。

7选择允许连接

进入“操作”设置页面后，选中 允许连接 选项（此选项表示允许符合规则的网络连接通过防火墙）。
选中后点击 下一步，进入规则应用场景设置。

8设置规则应用场景

在“配置文件”页面中，系统会列出规则可应用的场景（域、专用、公用），一般保持默认的全选状态即可（适用于大多数使用场景）。
确认选择后，点击 下一步，进入规则命名环节。

9命名规则并完成配置

在“名称”输入框中，填入该规则的名称，建议使用便于识别的名称，如 IIS服务端口允许规则 或直接输入 IIS。
若需要添加规则说明（如“开放IIS的80和443端口”），可在“描述”输入框中补充。
信息填写完成后，点击 完成 按钮，即可成功添加IIS服务的防火墙允许规则。
